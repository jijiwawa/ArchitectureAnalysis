# Protobuf Base64 NEON 优化

## 概述

优化 Protobuf upb 库中的 base64 解码函数 (`upb/json/decode.c`)，使用 ARM NEON SIMD 指令集加速。

## 原始实现分析

### 代码位置
- `upb/json/decode.c:572-637`
- 函数: `jsondec_base64()`

### 算法流程
```
输入: 4n 字节 base64 字符串
输出: 3n 字节二进制数据

主循环 (每次 4 字节):
  1. 查表获取 4 个 6 位值
  2. 合并为 24 位 (3 字节)
  3. 写入输出
```

### 性能瓶颈
1. **标量查表** - 每字节 1 次内存访问
2. **标量位移** - 无法并行处理多组数据
3. **逐字节写入** - 无法利用向量存储

## NEON 优化策略

### 1. 批量查表
- 一次加载 16 字节
- 使用 `vtbl` 指令向量化查表
- 处理 256 字节查找表需要分段

### 2. 并行位移
```
4 组并行:
  [c0,c1,c2,c3] -> [o0,o1,o2]
  [c4,c5,c6,c7] -> [o3,o4,o5]
  [c8,c9,c10,c11] -> [o6,o7,o8]
  [c12,c13,c14,c15] -> [o9,o10,o11]
```

### 3. 预期加速
- 理论: 4x (16 字节 / 4 字节)
- 实际: 预计 2-3x (查表开销)

## 注意事项

1. **查找表限制**: NEON vtbl 最多 64 字节，需要分段处理 256 字节表
2. **错误处理**: NEON 版本遇到非法字符需回退到标量版本
3. **尾部处理**: 长度非 16 倍数时使用标量处理
