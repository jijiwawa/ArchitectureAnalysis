## 5. 场景视图 (Scenarios / +1 View)

> 关注关键用例

### 5.1 场景一：实时直播编码

```
参与者: 摄像头 → 编码器 → 网络推流

流程:
┌──────────┐    ┌──────────────┐    ┌──────────┐
│ 摄像头   │───►│ uavs3e       │───►│ RTMP     │
│ 30fps    │    │ SpeedLevel=7 │    │ 推流     │
│ 1080p    │    │ threads-gop=8│    │          │
└──────────┘    └──────────────┘    └──────────┘

配置要点:
- SpeedLevel=7 (最快)
- threads-gop=8 (充分利用多核)
- RateControl=CBR (恒定码率)
- TargetBitRate=4000 (4Mbps)
```

### 5.2 场景二：视频会议

```
参与者: 多端 → MCU/SFU → 解码显示

流程:
┌──────────┐    ┌──────────────┐    ┌──────────┐
│ 本地摄像头│───►│ uavs3e       │───►│ WebRTC   │

│ 30fps    │    │ threads-wpp=4│    │          │
└──────────┘    └──────────────┘    └──────────┘

配置要点:
- SpeedLevel=6 (平衡速度/质量)
- threads-wpp=4 (移动端友好)
- RateControl=CBR
- TargetBitRate=1500 (1.5Mbps)
- 低延迟模式
```

### 5.3 场景三：移动端实时编码

```
参与者: 手机摄像头 → 编码 → 本地预览/上传

┌──────────┐    ┌──────────────┐    ┌──────────┐
```

## 6. 关键设计决策

### 6.1 RealTime vs Master 架构对比

| 决策点 | RealTime 分支 | Master 分支 | 理由 |
|--------|--------------|-------------|------|
| ALF | 移除 | 保留 | ALF 计算复杂度高，不适合实时 |
| 结构 | 单一 uAVS3lib | src/inc 分离 | 简化编译和集成 |
| 测试 | utest_gop.c | 无 | GOP 并行测试需求 |
| iOS | 完整 demo | 无 | 移动端实时场景 |

### 6.2 性能优化策略

| 策略 | 实现位置 | 效果 |
|------|----------|------|
| GOP 并行 | threadpool.c | 多核利用率提升 |
| WPP 波前并行 | slice.c | 帧内并行 |
| SIMD 优化 | x86/armv8/ | 2-5x 加速 |
| 速度等级 | SpeedLevel 5-7 | 可配置质量/速度 |
| 早退机制 | me.c, rdopt.c | 减少无效计算 |
